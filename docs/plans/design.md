# 家庭小药箱 - 产品设计文档

## 1. 项目概述

**项目名称**: 家庭小药箱
**项目类型**: 微信/支付宝小程序
**核心定位**: 轻量、免费、可靠的本地健康档案管理工具

**用户画像**: 一家三口/四口的小家庭

**核心价值**:
- 记录家庭成员的生病情况，形成健康档案
- 管理家庭药品，了解药品用途和有效期
- 智能识别减少手动输入
- 数据本地存储，可导出迁移

---

## 2. 功能模块

### 2.1 成员管理

| 功能 | 说明 |
|------|------|
| 添加成员 | 姓名、生日、血型、过敏源、备注 |
| 编辑成员 | 修改上述信息 |
| 删除成员 | 关联数据一并处理 |
| 成员头像 | 可选，支持拍照或相册选择 |

**字段定义**:
- `id`: 唯一标识
- `name`: 姓名
- `birthday`: 生日（YYYY-MM-DD）
- `bloodType`: 血型（A/B/AB/O/+/-）
- `allergies`: 过敏源列表（字符串数组）
- `avatar`: 头像路径
- `remark`: 备注
- `createdAt`: 创建时间

---

### 2.2 生病记录

| 功能 | 说明 |
|------|------|
| 添加记录 | 选择成员、日期、类型、症状、诊断 |
| 智能识别 | 拍照病例单，识别病症、诊断结果 |
| 关联药品 | 关联已添加的药品 |
| 按成员展示 | 列表页按成员筛选/分组 |

**字段定义**:
- `id`: 唯一标识
- `memberId`: 成员ID
- `date`: 生病日期
- `type`: 类型（就诊/自我判断）
- `diagnosis`: 诊断结果（AI识别/手动）
- `symptoms`: 症状描述
- `medicineIds`: 关联药品ID列表
- `photos`: 病例照片路径列表
- `doctorAdvice`: 医生建议
- `createdAt`: 创建时间

---

### 2.3 药品管理

| 功能 | 说明 |
|------|------|
| 添加药品 | 名称、用途、用法、过期日期、禁忌等 |
| 智能识别 | 拍照药品包装，识别药名、用途、用法、过期日期 |
| 照片存档 | 保留药品照片 |
| 过期检查 | 用药前主动检查是否过期 |
| 库存管理 | 可选，记录剩余数量 |

**字段定义**:
- `id`: 唯一标识
- `name`: 药品名称
- `purpose`: 用途/主治
- `usage`: 用法用量
- `purchaseDate`: 购买日期
- `expiryDate`: 过期日期
- `contraindications`: 禁忌人群
- `photos`: 药品照片路径列表
- `stock`: 库存数量
- `createdAt`: 创建时间

---

### 2.4 数据迁移

| 功能 | 说明 |
|------|------|
| 导出数据 | 导出为JSON文件（含Base64图片） |
| 导入数据 | 从JSON文件恢复数据 |
| 照片处理 | 支持导出/导入照片 |

---

### 2.5 智能识别（预留接口）

**设计原则**: 第一版只实现一个OCR服务，后期可切换

**接口预留**:
```typescript
interface OCRService {
  recognizeMedicalReport(imagePath: string): Promise<MedicalReportResult>;
  recognizeMedicine(imagePath: string): Promise<MedicineResult>;
}

interface MedicalReportResult {
  diagnosis: string;      // 诊断结果
  symptoms: string[];      // 症状描述
  doctorAdvice: string;   // 医生建议
}

interface MedicineResult {
  name: string;            // 药品名称
  purpose: string;         // 用途/主治
  usage: string;          // 用法用量
  expiryDate: string;     // 过期日期
  contraindications: string[]; // 禁忌人群
}
```

**第一版实现**: 使用阿里云免费OCR（或类似免费服务）

## 3. 技术架构

### 3.1 技术栈

| 层级 | 技术选择 |
|------|----------|
| 框架 | Taro 4.x + React |
| 样式 | Tailwind CSS / CSS Modules |
| 状态管理 | React Context / Zustand |
| 本地存储 | Taro DB（SQLite）或本地文件 |
| OCR/AI | 阿里云免费OCR（预留切换接口） |
| 图表/UI | 自定义组件 |

### 3.2 数据存储

- **本地存储**: 小程序自带的 SQLite 或文件存储
- **无服务器**: 完全客户端运行
- **数据迁移**: JSON 文件导出/导入

### 3.3 目录结构（预期）

```

├── components/          # 公共组件
├── pages/
│   ├── members/        # 成员管理
│   ├── illness/        # 生病记录

│   └── settings/       # 设置/迁移
├── services/
│   ├── storage.ts      # 本地存储服务
│   ├── ocr.ts          # OCR服务（预留）
│   └── export.ts       # 导出导入服务
├── types/              # TypeScript类型定义
└── utils/              # 工具函数
```

## 5. 页面设计

### 5.1 导航结构
- 底部Tab: 成员、生病记录、药品、设置
- 每个Tab对应一个独立页面

### 5.2 成员页面

**列表页：**
- 成员卡片列表（头像 + 姓名 + 年龄 + 血型 + 过敏源）
- 点击卡片进入编辑页面
- 右上角添加按钮

### 5.3 生病记录页面

**列表页：**
- 顶部成员Tab快速切换（一次只看一人）
- 简洁卡片列表：疾病名称 + 日期
- 点击进入详情
- 右下角快捷添加按钮

**详情页：**
- 疾病名称（输入框）
- 日期（日期选择器）
- 类型（就诊/自我判断 下拉选择）
- 描述（通用输入框，可记录任何想记录的信息）
- 关联药品（多选标签）
- 病例照片（添加按钮）

### 5.4 药品管理页面

**列表页：**
- 药品卡片列表（照片 + 名称 + 用途 + 过期状态）
- 过期状态直接显示：
  - 绿色 ✓ 正常
  - 橘色 ⚠️ 30天内过期
  - 橘红色 ❌ 已过期
- 左滑显示删除按钮
- 右上角添加按钮

**详情页：**
- 药品照片（点击添加）
- 药品名称（输入框）
- 用途/主治（输入框）
- 用法用量（输入框）
- 购买日期（日期选择器）
- 过期日期（日期选择器）
- 禁忌人群（输入框）
- 库存（数字输入）

### 5.5 设置页面

**设置首页：**
- 数据管理（点击进入子页面）
- OCR/AI 配置（点击进入子页面）
- 关于信息

**数据管理子页面：**
- 导出数据（导出为JSON）
- 导入数据（从JSON恢复）

**OCR/AI 配置子页面：**
- OCR 配置：服务商、AppCode
- AI 配置：服务商、API Key

## 6. 非功能性需求

| 需求 | 说明 |
|------|------|
| 免费 | 无任何付费功能，OCR使用免费额度 |
| 轻量 | 安装包小，启动快 |
| 可靠 | 数据本地存储，不丢失 |
| 可迁移 | JSON导出导入，换机无忧 |
| 隐私 | 数据仅存本地，不上传云端 |
| 简洁 | 页面简洁，交互直观 |
| 高效 | 快速切换，专注单人视角 |

## 7. 版本规划

### v1.0（快速可用）
- 成员管理（增删改查）
- 生病记录（按成员展示 + 详情）
- 药品管理（过期状态 + 左滑删除）
- 数据导出/导入
- OCR/AI 配置（预留）

- 家庭成员数据同步

---

## 8. 关键约束

1. **无服务器**: 纯客户端应用，不依赖后端服务
2. **免费优先**: OCR使用免费额度，不强制付费
3. **简洁高效**: 功能不求多，但每个功能都要好用
4. **用户主动**: 不推送提醒，用药前主动检查
